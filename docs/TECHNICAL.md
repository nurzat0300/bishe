### 2. 核心数据结构

#### LSA包
```python
LSAPacket:
  - packet_type: int (3 = LSA)
  - source_router_id: int
  - sequence_number: int
  - lsa_data: RouterLSA
  - hop_count: int (TTL)

RouterLSA:
  - router_id: int
  - sequence_number: int
  - timestamp: float
  - links: List[Link]
```

#### 路由表项
```
RouteEntry:
  - destination: int
  - next_hop: int

  - path: List[int]  # 完整路径
```

### 3. 消息流程

#### LSA泛洪过程
1. 路由器定期（每5秒）生成LSA
2. 序列号自增，避免重复
3. LSA通过UDP广播到所有邻接路由器
4. 邻接路由器接收LSA，检查序列号
5. 若为新LSA，更新拓扑数据库并继续转发
6. 触发SPF计算更新路由表

#### 故障检测过程
1. 路由器定期（每0.3秒）发送心跳消息
2. 邻接路由器接收心跳，重置失效计数
3. 若连续3个心跳间隔未收到（>0.9秒），判定邻接故障
4. 立即生成新LSA，标记链路DOWN
5. 转发LSA，触发全网重新收敛

### 4. 算法实现

#### Dijkstra算法
- 使用二叉堆优化
- 时间复杂度：O((V+E)logV)
- 适合小到中等规模网络

#### 新算法
- 采用分层优先队列
- 减少无效松弛操作
- 期望在某些场景下性能更优

### 5. 多线程管理

每个路由器实例包含5个独立线程：

| 线程名 | 功能 | 周期 |
|--------|------|------|
| recv | 接收UDP消息 | 连续 |
| send | 发送队列中的消息 | 连续 |
| process | 处理接收的消息 | 连续 |
| lsa | 定期发送LSA | 5秒 |
| heartbeat | 发送心跳和检测超时 | 0.3秒 |

```python
is_new_lsa(source_id, seq_num):

    return True

```

### SPF计算（Dijkstra）

```
1. 初始化距离表：distances[source] = 0
2. 创建优先队列，加入源点
3. 当队列非空：
   a. 取出距离最小的节点
   b. 标记为已访问
   c. 遍历所有邻接节点
   d. 若找到更短路径，更新距离和路径
   e. 新节点入队
```

## 已知限制

1. 拓扑规模：当前实现支持最多255个路由器
2. 通信方式：仅支持本地UDP（单机/局域网）
3. 链路成本：整数值（1-65535）
4. 算法改进：新算法仍需与论文原文进一步对齐

## 扩展建议

1. **支持分布式部署**：使用实际网络而非本地UDP
2. **集成更多算法**：SPF变体、ECMP等
3. **增强故障检测**：使用BFD精确检测
4. **支持IPv4/IPv6**：扩展地址家族
5. **性能优化**：使用C++实现核心算法
