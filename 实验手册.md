## 1. 系统简介

本系统是一个**链路状态路由协议（Link State Routing Protocol）的分布式仿真平台**，用于计算机网络课程教学。系统模拟了 OSPF 协议的核心机制，包括邻居发现、LSA 洪泛、SPF 最短路径计算和路由表生成。

### 核心功能

| 功能 | 说明 |
|---|---|
| 分布式路由器仿真 | 每个路由器独立运行，通过 UDP 进行通信 |
| Hello 协议 | 心跳检测与邻居发现 |
| LSA 洪泛 | 可靠的链路状态通告传播机制 |
| SPF 计算 | 基于 Dijkstra 算法的最短路径优先计算 |
| 多算法对比 | 集成 7 种最短路径算法的性能对比 |
| 算法动画 | Dijkstra 算法的分步执行可视化 |
| 故障模拟 | 链路故障与恢复的动态演示 |
| 拓扑切换 | 支持 8/12/20/50 节点拓扑一键切换 |

### 适用课程

- 计算机网络（路由协议章节）
- 数据结构与算法（图论最短路径）
- 分布式系统（一致性、洪泛机制）

**用途：** 运行多种最短路径算法的性能对比测试。

**操作步骤：**
1. 选择**源路由器**
2. 设置**测试轮数**（建议 10-50 轮，轮数越多结果越稳定）
3. 点击 **执行对比计算**

**对比算法：**

| 算法 | 类型 | 时间复杂度 |
|---|---|---|
| Dijkstra | 贪心 + 优先队列 | O(E log V) |
| 新型算法 | 分层桶优化 | O(E log V) |
| Bellman-Ford | 动态规划 | O(VE) |
| SPFA | 队列优化 Bellman-Ford | O(kE) |
| Prim (MST基线) | 最小生成树 | O(E log V) |
| Floyd-Warshall | 全源动态规划 | O(V³) |
| A* | 启发式搜索 | O(E) |

**结果指标：**
- 平均/最小/最大耗时 (ms)
- 平均访问节点数
- 平均迭代次数
- 平均松弛操作次数

## 8. 如何添加新算法

添加一个新的最短路径算法只需修改 **3 个文件**。以下以添加"双向 Dijkstra (Bidirectional Dijkstra)"为例。

### 步骤 1：编写算法类

在 `src/algorithms/` 下新建文件，例如 `bidirectional_dijkstra.py`：

```python
"""
双向Dijkstra算法 - 从起点和终点同时搜索
"""
import heapq
import time
from typing import Dict, List, Tuple


class BidirectionalDijkstra:
    """双向Dijkstra最短路径算法"""

    def __init__(self):
        self.computation_time = 0.0
        self.visited_nodes = 0
        self.iterations = 0

    def calculate_shortest_paths(
        self,
        source: int,
        graph: Dict[int, List[Tuple[int, int]]],
    ) -> Dict[int, Tuple[int, List[int]]]:
        """
        计算从 source 到所有其他节点的最短路径。

        Args:
            source: 源节点 ID
            graph: 邻接表 {node: [(neighbor, cost), ...]}

        Returns:
            {destination: (total_cost, [path_nodes])}
        """
        start_time = time.perf_counter()
        self.visited_nodes = 0
        self.iterations = 0

        # ---- 你的算法实现 ----
        # 这里用标准 Dijkstra 代替，请替换为实际双向搜索逻辑
        distances = {source: 0}
        paths = {source: [source]}
        visited = set()
        pq = [(0, source)]

        while pq:
            self.iterations += 1
            curr_cost, curr_node = heapq.heappop(pq)
            if curr_node in visited:
                continue
            visited.add(curr_node)
            self.visited_nodes += 1

            if curr_node in graph:
                for neighbor, cost in graph[curr_node]:
                    new_cost = curr_cost + cost
                    if neighbor not in distances or new_cost < distances[neighbor]:
                        distances[neighbor] = new_cost
                        paths[neighbor] = paths[curr_node] + [neighbor]
                        heapq.heappush(pq, (new_cost, neighbor))

        self.computation_time = time.perf_counter() - start_time

        result = {}
        for dest in distances:
            if dest != source:
                result[dest] = (distances[dest], paths.get(dest, []))
        return result

    def get_statistics(self) -> Dict:
        """返回算法统计信息（必须实现）"""
        return {
            'algorithm_name': 'BidirectionalDijkstra',
            'computation_time': self.computation_time * 1000,  # 毫秒
            'visited_nodes': self.visited_nodes,
            'iterations': self.iterations,
        }
```

## 10. 常见问题

### Q1：启动报错 "DLL load failed" 或 "找不到 Qt 平台插件"

**原因：** PyQt5 与系统不兼容或未正确安装。

**解决：**
```bash
pip uninstall PyQt5 PyQt5-sip -y
pip install PyQt5==5.15.9 PyQt5-sip==12.13.0
```

### Q2：防火墙提示网络访问

**原因：** 路由器之间使用 UDP 进行通信（仅本地 127.0.0.1），防火墙可能拦截。

**解决：** 点击"允许访问"，或在防火墙中添加 Python 的例外规则。

### Q3：端口被占用

**原因：** 上次程序未正常退出，端口仍被占用。

**解决：**
- Windows：打开任务管理器，结束所有 `python.exe` 进程
- 或修改 `config/network_topology.json` 中的端口号

### Q4：切换到 50 节点拓扑后界面卡顿

**原因：** 50 节点拓扑的路由表渲染开销较大。

**解决：** 这是正常的，系统已自动启用汇总模式。如仍感卡顿，可以：
- 减少同时打开的标签页
- 调大统计刷新间隔（修改 `src/ui/main_window.py` 中的 `timer.start(2000)` 为更大的值）

### Q5：如何修改心跳检测间隔？

修改 `src/core/constants.py`：
```python
HEARTBEAT_INTERVAL = 0.8   # 心跳发送间隔（秒）
HEARTBEAT_TIMEOUT = 3      # 连续超时次数阈值
```

### Q6：如何在不同机器上分布式运行？

1. 修改配置文件中的 `listen_ip` 为各机器的实际 IP 地址
2. 确保各机器之间网络互通，防火墙开放对应 UDP 端口
3. 在所有机器上同步配置文件
4. 分别启动各机器上的程序
